# Правила оформления отчетов для typst

## План написания отчета:
- Проанализируй отчет другого студента, он лежит в файле example.pdf
- Составь подробный план этого отчета и провалидируй его об меня
- Запроси скриншоты приложения, которые тебе понадобятся, напиши как их назвать.
- Напиши отчет по составленному плану, используя предоставленные скриншоты
- Следуй правилам оформления ниже

## Глобальные правила

- не используй заголовки третьего уровня, четвертого и т.д. уровней
- не хардкодь нумерацию никогда и нигде ни в списках, ни в заголовках. Используй "-" для маркированных списков и "+" для нумерованных

## Работа с рисунками
для вставки изображений используй #picture
#picture(
    path: img("path") # путь к .png картинке относительно report/imgs/
    caption: [Текст пояснения] # Пояснение к рисунку. пиши что изображено на рисунке. Не пиши что-то вроде "Рисунок N", это добавится автоматически
    width: 70% # используй, чтобы рисунок не занимал слишком большую часть страницы
) <ref_name> # всегда добавляй ссылку.

Для ссылки на рисунок используй #r(<ref_name>) – всегда добавляй ссылки на все рисунки.

## Работа с листингами
Для вставки листинга файла используй
#listing(
  file: "../lib/main.dart", # путь к файлу относительно файла с отчетом
  caption: [Текст пояснения], # подпись к листингу
  from: 1, # с этой строки
  to: 13 # по эту строку
) <ref_name>

Для обычных блоков кода тоже используй
#listing(
  caption: [Текст пояснения], # подпись к листингу
  body: raw("Код") # Тут пишешь код, очень важно, чтоб был raw
) <ref_name>



На каждый листинг должна быть ссылка. Ссылайся с помощью #r(<ref_name>).
На классы и виджеты всегда делай полные листинги.
